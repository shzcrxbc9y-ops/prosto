## GITHUB_TOKEN - пошаговая инструкция

**GITHUB_TOKEN** нельзя "узнать" - его нужно **создать** в настройках GitHub. Вот как это сделать:

### Шаг 1: Откройте страницу создания токена

1. Перейдите по ссылке: **https://github.com/settings/tokens**
   
   Или вручную:
   - Войдите в GitHub
   - Нажмите на ваш аватар (правый верхний угол)
   - Выберите **Settings**
   - В левом меню найдите **Developer settings**
   - Нажмите **Personal access tokens**
   - Выберите **Tokens (classic)**

### Шаг 2: Создайте новый токен

1. Нажмите кнопку **"Generate new token"**
2. Выберите **"Generate new token (classic)"**

### Шаг 3: Заполните форму

**Note (Название токена):**
- Введите любое имя, например: `Vercel Data Saver` или `Netlify Data Saver`
- Это просто для вашего удобства, чтобы помнить, для чего токен

**Expiration (Срок действия):**
- Выберите срок действия:
  - **30 days** - токен будет работать 30 дней
  - **90 days** - токен будет работать 90 дней
  - **No expiration** - токен будет работать всегда (менее безопасно, но удобнее)

**Select scopes (Выберите права):**
- Найдите и отметьте галочкой: **`repo`**
- Это даст полный доступ к репозиториям (чтение и запись)
- При отметке `repo` автоматически выберутся все подразделы

### Шаг 4: Создайте токен

1. Прокрутите вниз
2. Нажмите зеленую кнопку **"Generate token"**

### Шаг 5: СКОПИРУЙТЕ ТОКЕН СРАЗУ! ⚠️

**ВАЖНО:** GitHub покажет токен **ТОЛЬКО ОДИН РАЗ**!

Токен будет выглядеть примерно так:
```
ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

1. **Скопируйте токен полностью** (нажмите на иконку копирования или выделите и Ctrl+C)
2. **Сохраните его в безопасном месте** (например, в блокноте или менеджере паролей)
3. **НЕ ПУБЛИКУЙТЕ** токен в открытом доступе (GitHub, форумы, чаты)

Если вы закроете страницу без копирования - токен будет потерян, и придется создавать новый!

## Как использовать токен

### Для Vercel:

1. Откройте ваш проект в [Vercel Dashboard](https://vercel.com/dashboard)
2. Перейдите: **Settings** → **Environment Variables**
3. Нажмите **"Add New"**
4. Заполните:
   - **Key**: `GITHUB_TOKEN`
   - **Value**: вставьте скопированный токен
   - **Environment**: выберите все (Production, Preview, Development)
5. Нажмите **"Save"**

### Для Netlify:

1. Откройте ваш проект в [Netlify Dashboard](https://app.netlify.com)
2. Перейдите: **Site settings** → **Environment variables**
3. Нажмите **"Add a variable"**
4. Заполните:
   - **Key**: `GITHUB_TOKEN`
   - **Value**: вставьте скопированный токен
   - **Scopes**: выберите **"All scopes"** или **"Production, Deploy previews, Branch deploys"**
5. Нажмите **"Save variable"**

## Проверка работы

После добавления переменных:

1. **Пересоберите проект** в Vercel/Netlify
2. Откройте ваш сайт
3. Введите данные в форму
4. Нажмите "Сохранить"
5. Проверьте файл `data.txt` в GitHub: https://github.com/shzcrxbc9y-ops/prosto

Данные должны появиться в файле через несколько секунд!

## Если токен потерян или скомпрометирован

Если вы потеряли токен или он был скомпрометирован:

1. Перейдите: https://github.com/settings/tokens
2. Найдите ваш токен в списке
3. Нажмите на иконку корзины (Delete) справа от токена
4. Подтвердите удаление
5. Создайте новый токен по инструкции выше
6. Обновите переменную `GITHUB_TOKEN` в Vercel/Netlify

---

## Безопасность

⚠️ **Важные правила безопасности:**

- ✅ Храните токен в секрете
- ✅ Используйте переменные окружения (не вставляйте в код)
- ✅ Не публикуйте токен в GitHub, форумах, чатах
- ✅ Регулярно обновляйте токены (особенно если срок истек)
- ✅ Удаляйте неиспользуемые токены

❌ **НЕ ДЕЛАЙТЕ:**
- Не коммитьте токен в Git
- Не отправляйте токен по email/мессенджерам
- Не используйте один токен для всех проектов (лучше создавать отдельные)
